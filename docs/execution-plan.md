## Phase 2 — Integrative Review Execution
**Duration:** Weeks 3–5  
**Goal:** Execute the literature search and build the coverage matrix with real data.

### Search Protocol

**Databases:** Scopus (primary), Web of Science, ERIC, IEEE Xplore  
**Search strings (Boolean):**
```
("evaluation framework" OR "assessment model" OR "quality framework") 
AND 
("educational technology" OR "e-learning" OR "CALL" OR "language learning platform" OR "EdTech" OR "LMS" OR "digital learning platform")
AND
("higher education" OR "university" OR "tertiary")
```
**Filters:** 2001–2025 | English | Article or Book Chapter | Peer-reviewed  
**Exclusion criteria:** Empirical papers without framework component | Corporate/non-HE platforms | Gray literature

|---|---|---|---|

| 2.3 | Full-text review of shortlisted papers | Read and code | ~20–30 included frameworks |
| 2.4 | Extract: dimensions, indicators, assessment criteria per framework | Extraction form (Excel/Sheets) | Framework extraction table |

## Phase 5 — Theoretical Validation (CFA Phase 7–8)
**Duration:** Weeks 14–15  
**Goal:** Execute desk-based theoretical cross-validation; iterate on framework.

### Validation Procedure

**CFA Phase 7 — Theoretical Cross-Validation:**
1. Take each existing framework from the coverage matrix
2. Check whether the proposed framework's dimensions close all identified gaps
3. Check for redundancy: does any existing framework already cover what this paper claims is novel?
4. Check logical coherence: do all 8 propositions follow from their stated theoretical bases without contradictions?

**CFA Phase 8 — Rethinking & Revision:**
1. Document any inconsistencies found in Phase 7
2. Revise indicators, propositions, or framework structure as needed
3. Final cross-mapping: proposed framework vs. D&M constructs (ensure alignment)

### Tasks

| # | Task | Output |
|---|---|---|
| 5.1 | Systematic cross-check of coverage matrix vs. proposed framework | Validation log |
| 5.2 | Logical consistency audit of P1–P8 | Annotated propositions |
| 5.3 | Check for novelty: is any dimension fully covered by an existing single framework? | Novelty confirmation |
| 5.4 | Revise draft based on Phase 7–8 findings | Revised draft v0.2 |

### Quality Checkpoint
- [ ] Coverage matrix shows proposed framework is non-redundant
- [ ] All propositions internally consistent (no circular reasoning)
- [ ] Validation section of paper explains *what was checked* and *what was confirmed*
